# Day 10 — 7 July 2026 (evening session)

## Sound, body language, phones, and safety

Simon's observations became the work order — plus one constitutional amendment and one big sensory addition.

## What got built (v0.10.0 — 137 KB, 122/122 checks green)

**The world makes sound now.** There were no sounds before; now everything is **synthesised live from oscillators and filtered noise — no audio files, zero dependencies**: wind that rises with the breeze and howls in storms, rain patter, rolling thunder-rumbles, tiny two-note **chirps when kith speak** (each in its own register, derived from its voice genome), and — the crown — **the singer's actual melody**: slow pentatonic notes drawn from its genome, played while it sings against the storm. Browsers demand a user gesture before audio, so the world stays politely silent until first touch (and now *resumes* correctly on the next touch if the browser parked the audio — a real-world bug caught during verification). A 🔊/🔇 button lives in the topbar; the preference sticks per device.

**Body language.** Kith now *sit* when dozing, *hunker low* when sheltering, *lean in* when sipping nectar, and *waddle* as they walk; a contest's loser flees with a "!" over its head. Plants already swayed — storms quicken them. Small, cheap, and the field finally reads as alive at a glance.

**Speech overlap fixed** (Simon's #1): bubbles stagger height and drift by identity, so close talkers no longer collide — verified live at three distinct heights.

**Phones** (Simon's #2): the noscript message now explains the real problem (Android's built-in HTML viewer has no JavaScript) with the fix (Files app → Open with → Chrome). Better: the build now emits **`docs/index.html`** — the same file — so the moment Simon flips one switch (repo Settings → Pages → deploy from `main`, folder `/docs`), the game lives at a URL any phone can visit. A hosted copy is just another copy; nothing about the mission changes.

**Safety** (Simon's #9): the norm is now taught in-game and in the playtest guide — **"open files you made; merge files you receive"** (merging only parses data and can never run anything) — and a 24 MB cap rejects hostile giants before parsing (tested: rejected in under half a second).

**The constitution amended** (Simon's size question): principle 1 now reads *light enough to travel* — the file must pass through a plain email attachment (≤ 20 MB) and open on weak hardware; within that, **richness beats minimalism**. Zero dependencies remains the unbreakable part. The build enforces the ceiling.
